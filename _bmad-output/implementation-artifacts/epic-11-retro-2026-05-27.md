### 3. Pre-create catalog gate missed `DF-X-SY` references added during dev (2 stories in a row)

Story 11.2 + Story 11.3 BOTH had cross-LLM reviewers catch `DF-X-SY` inline references that landed AFTER the pre-create gate ran. The pre-create gate ran correctly for known-at-create-time carry-overs (C75 + C76 at Story 11.2; no new ones at Story 11.3 create). But docstring + comment additions during dev added inline `DF-11.2-S3` (in `copilot_cli.py:198`) + `DF-11.3-S1` (in 3 adapter files + stability-surface) without bumping the catalog.

**Lesson:** the pre-create gate is a story-author-time check; it doesn't extend to dev-time additions. The cross-LLM review caught both in Epic 11. The "review-time catalog enforcement" sub-pattern (see "What worked" #3) closes the loop — but it's reactive (catches after the dev work is done), not proactive. The proper fix would be a `pre-commit` hook or a CI grep step.

### 4. Codex CLI adapter's `_VERSION_RE` dead-code constant — false confidence at spec-time

Story 11.1 spec drift check D-10 explicitly added `_VERSION_RE = re.compile(r"^codex-cli\s+(\d+\.\d+\.\d+)")` as a Phase-1 decision. The first dev-pass actually wired the constant. Cross-LLM review caught (both reviewers, MED) that the base `_SEMVER_RE.search()` handles the `codex-cli ` prefix natively — the override was dead code. Removed in v0.3.0.

**Lesson:** spec-time decisions about regex overrides need empirical validation against the base helper's behavior BEFORE the override lands. Story 11.2 + 11.3 didn't repeat the pattern (NO `_VERSION_RE` constants), so the lesson did propagate via `feedback_cross_story_upstream_lesson_propagation`.

### 5. Story 11.2's `run()` override pattern is the first post-hoc-disk-trace adapter

The architectural wrinkle in Story 11.2: Copilot writes events to `~/.copilot/session-state/{uuid}/events.jsonl` rather than stdout. The base `SubprocessAdapter.run()` template-method assumes streaming-stdout. Story 11.2 overrides `run()` to do `spawn → wait → glob newest dir → read events.jsonl → finalize`.

This adds 2 invariants (session-state-dir race + post-exit flush race) that are documented in the class docstring + C77 carry-over. Phase-2 work to fix properly: read `session.start.sessionId` from stdout pre-`proc.wait()` then locate the dir by name + add retry loop for post-exit flush.

**Lesson:** the `SubprocessAdapter` ABC assumes streaming-stdout; out-of-process disk-trace adapters need override + invariant documentation. Phase-2 may want to factor a `PostHocTraceAdapter` sibling ABC if a 2nd post-hoc adapter emerges.
