## What went well

1. **Adversarial review pattern earned its cost three times.**
   - Story 0.1 review caught: `request_handlers` private-attribute coupling concerns; `mcp_coverage` 3-state semantic ambiguity (became D1 trust-floor amendment); stdio path "not actually a real observer" (became D2 wrapper-injection rework).
   - Story 0.2 review caught: `baseline_leak` math undermining the headline zero-leaks claim; `startup_latency_ms` mislabeled (measures lifetime, not startup); the atexit-on-SIGKILL gap that surfaced the D2.4 finding (Python's default SIGTERM ALSO bypasses atexit — much stronger than the original spike anticipated).
   - Story 0.3 review caught: fabricated `§AC-0.2.3.d` citation; AC-0.3.4 stale downstream-story refs in epics.md that I'd marked vacuously-satisfied; word-count violation (claimed 194, actual 260); architecture.md line numbers drifted by ~13 after Step-4 delta insertion.
2. **Cross-LLM family diversity is load-bearing for documentation.** Story 0.3's two-reviewer setup (Claude Opus + Codex CLI): Codex marked NO-GO and caught the AC-0.3.4 blocker that Claude (same family as the writer) missed entirely. Same-family review has correlated blind spots that cross-family review surfaces.
3. **D5 reproduction surfaced a real fragility through 3-agent diversity.** All three reproducers independently flagged the smoke-matrix harness back-to-back contamination — a real bug my internal review rounds missed. The finding was documented honestly (Option A: downgrade headline from "45/45 clean" to "9/9 cells clean in isolation") rather than buried.
4. **Honest framing held when reality forced a downgrade.** When Story 0.2's matrix harness failed reproduction, the findings doc absorbed the change in §AC-0.2.1 without rewriting the verdict substance. The verdict (Listener v3 primary + auto-installed SIGTERM handler + SIGKILL unrecoverable) survived intact because it was supported by the standalone probes, not just the headline matrix run.
5. **Decision records are auditable.** Every ratified ADR cites the source spike findings doc + the empirical evidence section. ADR-004's handler-wrap pattern, ADR-016's trust-floor semantic, ADR-018's verbatim ratification — all traceable to spike outputs that future reviewers can re-run.

## What challenged us

1. **Methodology error: parallel-launching 3 reproducer agents in the same workspace caused them to fight over shared state.** Codex + Copilot smashed each other's `measurements/` dir + competed for the pabotlib port. Most of their NO-GO verdicts were partly artifacts of my methodology, not the spike's substance. **Lesson: serialize reproducers when they share workspace state.**
2. **Self-review blind spots are real even for documentation.** Story 0.3 was supposed to be "just docs" — I lifted amendment text from findings docs into MADR templates. The first pass had multiple citation-drift bugs that two independent reviewers found in ~10 minutes. The writer's perspective is unreliable for accuracy checks.
3. **Honesty-failures crept in despite explicit honest-framing intent.** Examples: "194 words" claim was unverified before commit (actual 260); `§AC-0.2.3.d` citation was a made-up reference (the section doesn't exist); "45/45 clean" headline was robust under quiet conditions but fragile under sustained pabot load. **Lesson: numeric claims must be machine-verified before commit, not eyeballed.**
4. **macOS gap accepted via D2.1 architect waiver.** Defensible given no macOS hardware, but represents a known production-readiness gap that surfaces as a Phase-1.5 carry-over.
5. **rf-mcp substituted; not real binary.** No git access in spike env. Substitute mimics qualitative profile but doesn't validate real-world startup timing. Promoted from footnote to primary risk in ADR-018 during review.
6. **Matrix harness fragility documented but not fixed.** Cross-cell contamination + pabot back-to-back races known. Production code (Story 1b.1) won't use the harness; acceptable trade-off, but might surface in future Phase-1 spike work.
7. **LLM "spike-time" is fundamentally compressed.** Story 0.1 = 5-day human spike → ~3h LLM work; Story 0.2 = 3-day → ~3h. D5 reproduction partially compensates by surfacing what the compression missed, but isn't a 1:1 substitute for sustained architect curiosity.

## Key insights

- **Adversarial review converts "ship a spike" into "ship a calibrated spike."** Real cost (~30-60 min per round + architect review) earns real value (each round caught compounding defects).
- **Citation drift is the subtlest failure mode in documentation-heavy work.** Quick to write, hard to spot, expensive to fix after compounding into downstream stories.
- **Cross-LLM family diversity matters MORE for documentation than for code.** Code has tests; docs don't. External readers catch what writers can't see.
- **"Phase-1 carry-over" framing is honest but loose.** Tracking via `deferred-work.md` keeps risks visible; without explicit Phase-1.5 owners, they can slip.

## Action items

Per Many's retrospective decision (2026-05-17), three lessons are captured as project-level norms:

1. **Adversarial review pattern is project standard.** Every story gets a `/bmad-code-review` cycle. Review configurations should mix model families (Claude + Codex/Copilot/Sonnet, not Claude alone). Documented in `deferred-work.md` under "Project methodology norms". **Owner: project methodology. Applies starting Story 1a.1.**
2. **Numeric claims must be machine-verified before commit.** Word counts (`wc -w`), file counts, leak counts, percent margins, line numbers — pipe through a script before writing the claim. **Owner: any author writing findings docs, ADRs, or claim-heavy artifacts. Applies starting Story 1a.3 (which ratifies 14 more ADRs and is the highest-volume documentation work in Phase 1).**
3. **Serialize multi-agent reproductions when they share workspace state.** Never parallel-launch reproducers that write to the same `measurements/` or run concurrent processes against the same workspace. **Owner: D5-style reproduction protocol. Applies to any future spike reproduction.**
