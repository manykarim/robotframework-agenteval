### 3. Empirical SDK probe pattern load-bearing for both Phase-2 adapters

Both Phase-2 adapters shipped with `usage` shape bugs that ONLY empirical SDK probes caught:

- **Story 10.1 HIGH-1** (Claude CLI catch): `claude-agent-sdk`'s `ResultMessage.usage` is a `dict`, not an attribute-bearing object. Pre-patch `getattr(result_msg.usage, "input_tokens", 0)` returned `0` on every real run — fake-test masked by `_FakeUsage` shim that had attributes. Without Claude's `dataclasses.fields(ResultMessage)` probe, every live run would have silently reported `Usage(0, 0, 0)`.
- **Story 10.2 manual probe** (operator catch, v0.4.0): `openai-agents`'s `RunResult` has NO top-level `usage` attribute. Canonical path is `result.context_wrapper.usage`. Same fake-test masking pattern.

The fix on both sides was a factored helper (`_extract_usage` / `_project_agents_usage`) with defensive isinstance branching + a `test_extract_usage_handles_both_shapes` regression-guard.

**Lesson:** Phase-2 vendor SDKs are pre-1.0 and ship with non-obvious response shapes. The fake-test pattern + the SDK's typing-hint imprecision combine to mask zero-token bugs. `feedback_listener_hook_api_surface_empirical_check` (Epic 8 retro NEW norm) is now validated across 3 distinct surfaces: Listener v3 hook (Story 8a.2 origin), Claude Agent SDK (Story 10.1), OpenAI Agents SDK (Story 10.2). **Confirmed pattern; not a candidate anymore.**

### 4. Mock-incompatible `mcp_coverage` honesty default landed on both adapters

Both adapters mirror the post-HIGH-2-patch contract: non-empty `mcp_servers` returns `mcp_coverage="external_mixed"` until per-adapter `HostedMcpObserver` wiring lands (C68 / C69). This is the **ADR-016 L59 safer-default rule operating as designed** — the alternative (optimistic `hosted_in_process` on any non-empty MCP attachment) would have silently inflated coverage metrics for every Phase-2 user running with MCP. Caught on Story 10.1, applied UPSTREAM on Story 10.2.

The kilo/minimax MED-2 catch on Story 10.1 retroactively corrected **16 citation references** from `ADR-A6 L384` (renumbered + line-number bogus) to `ADR-016 L59` across both `claude_agent_sdk.py` (9) + `openai_agents.py` (7) + `docs/phase-1-5-carry-overs.md` (1). The renumbering trail was months old + escaped Claude CLI's review entirely. This **validates `feedback_citation_drift_first_class` (Epic 1a retro NEW norm) at N=44+ consecutive uses** — and the third-LLM-family fallback is the mechanism that surfaces drift Claude misses.
