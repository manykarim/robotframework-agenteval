## Story-by-Story Pattern Density

Five emergent patterns compounded across the 4 stories:

1. **`_FR59Tier1SetupFailureError` private intermediate** — Story 2.2 refactor extracted shared `__init__` + `__str__` from Story 2.1's `InvalidSkillFrontmatterError`. Story 2.3 inherited it for both MCP leaves. 5 setup-failure leaves now share the FR59 4-line shape via one private parent. The refactor is behavior-preserving (Story 2.1 tests unchanged) — proves the "ship Phase-1; refactor when the second case lands" path works.
2. **DynamicCore composition collision-prevention norm** — Story 2.2 HIGH-1: `Get Frontmatter` collision between Skills + Subagents under DynamicCore last-wins silently shadowed Story 2.1's validation. Resolution ratified at Story 2.2 + applied preemptively at 2.3 (MCP excluded) + 2.4 (conformance test enforces it). The runtime collision-detector in `AgentEval._build_components` + the exclusion-based registry are now load-bearing infrastructure.
3. **RFC 6901 JSON Pointer `field_name` convention** — Story 2.2 D-D ratified; Story 2.3 inherited for both MCP leaves + caught the `*` wildcard pseudo-pointer bug (3-way HIGH) when extending. Single source of truth for nested-JSON path semantics.
4. **Phase-1 carve-out registries** — Story 2.1 `Should *` keyword carve-out (architecture L825-839); Story 2.3 `.mcp.json:tools` declarative-extension carve-out. TWO different kinds of carve-out (anti-pattern exception vs. runtime-vs-declarative-data source). Both ratified explicitly.
5. **3-shared-parser-pattern inheritance** — `utf-8-sig` BOM strip + `.rstrip()` column-0 delimiter + `exc.problem` YAML one-liner. Story 2.1 fixed all 3 in code-review (Blind + Edge-cases + Codex caught); Story 2.2 + 2.3 inherited transitively without re-discovery. **First time** in this project we've seen a pattern propagate cleanly across 3 sub-libraries without re-bug.

## What went well

1. **Cross-LLM review pattern is now load-bearing at scale.** 16 consecutive STAR catches across Epic 2's 4 stories. Without the 4-reviewer pair, Story 2.2's silent `Get Frontmatter` collision + Story 2.4's ungated conformance step would have shipped — both HIGH-severity correctness gaps invisible to single-LLM-family review.
2. **3-way agreement is the strongest signal.** Story 2.2's collision: Blind + Edge-cases via behavioral probe caught it; Codex MISSED it (set-deduplicated `Counter` probe). Story 2.3's `*` wildcard pseudo-pointer: 3-way (Blind + Edge-cases + Codex). Story 2.4's ci.yml-missing-step: 3-way too. **Codex isn't always the STAR catcher** — sometimes the lane-specific Claude reviewer catches what Codex's pattern-matching misses.
3. **Pre-create-story drift check stayed at 100% adoption.** 4/4 stories, 5 drifts each, all resolved pre-authoring. 14 consecutive uses since Story 1a.3.
4. **Phase-1 carve-out registry pattern handles the AssertionEngine deferral cleanly.** Story 2.1's `Should Be Valid Frontmatter` (Phase-1 plain `@keyword` per ADR-022 catalog row) shipped without PRD amendment — just an architecture L825 carve-out + a conformance-test allowlist. Story 2.4 added an AC-SIMPLICITY-02b paired-getter test that trivially passes Phase-1 but becomes load-bearing in Epic 4+. Future-proof done right.
5. **Test infrastructure scaled.** 24 conformance fixtures + 17 integration tests + 13 RF integrations + 4 sub-library `test_robot_integration.robot` suites. Static-inspection schema is parallel to Story 1b.5's adapter schema — no harness churn.

## What challenged us

1. **CI fake-green vectors keep emerging.** Story 2.4 HIGH-1 was the worst: `pytest tests/conformance -q` step was MISSING from `ci.yml`; the comment claimed it was "already an existing step." 22 conformance cases + the un-skipped SIMPLICITY-02 introspections were UNGATED on PR. Three reviewers (Blind + Auditor + Codex) caught it independently. **Second time** the project has shipped a CI-config bug that only cross-LLM review caught (first was Story 1a.2 fake-green `--ignore=tests/unit/conventions`). `feedback_ci_log_forensics` norm continues to load-bear.
2. **Test-ordering fake-green at Story 2.1.** `tests/unit/conventions` failed STANDALONE due to a circular import in `src/AgentEval/skills/__init__.py:27` (eager `from AgentEval.skills.library import SkillsLibrary`). Full pytest run masked it via test-ordering. Auditor + Edge-cases + Codex agreed (3-way HIGH). Lesson now part of the project pattern: sub-library `__init__.py` files contain ONLY module docstrings; no re-exports.
3. **PRD ambiguity surfaced repeatedly.** Story 2.2 (FR3 schema over-constraint) + Story 2.3 (FR5 `name` is outer KEY) + Story 2.4 (AC-CONFORMANCE-01 paraphrase) all required PRD amendments mid-implementation. **PRD prose is the canary** for un-derived ambiguities — but Epic 2's amendments were always tightening / clarifying ratified contracts, never material scope changes.

## Key insights

- **The 16-consecutive-STAR-catch streak is structurally load-bearing, not anecdotal.** Two HIGH-severity correctness gaps were caught that single-LLM review would miss. 4-reviewer cross-LLM review is now the project's primary quality gate.
- **The Phase-1 carve-out registry pattern is becoming a primary contract mechanism.** Stories 2.1 + 2.3 + 2.4 each added a carve-out. The pattern preserves PRD-ratified contracts while letting Phase-1 ship pragmatic subsets that don't require PRD amendment.
- **Pattern propagation works when the lesson is encoded in code, not just docs.** Story 2.1's 3 parser fixes propagated to 2.2 + 2.3 without re-bug because the patterns were in the code template. Story 2.2's runtime collision-detector enforced the exclusion-based registry — Story 2.3 didn't need a separate test to validate it.
- **Codex pattern-matches; Claude reviewers probe.** Codex caught Story 2.2's FR3 schema over-constraint (no Claude reviewer did), but MISSED Story 2.2's keyword collision (the behavioral probe Codex chose was set-deduplicated; only Blind + Edge-cases re-derived from runtime behavior). Lesson: review prompts MUST specify behavioral probes for invariants that the type system / introspection can't catch.
